Email munger script instructions

Encoding an e-mail address with JavaScript is a simple way to protect it from being harvested by spam bots. The email munger script places Unicode characters in place of alphanumerics and punctuation in the anchor link. To use it, construct mailto: hyperlinks like this:

web-info@ webcentre.unimelb.edu.au

The text part of the link has a deliberate spacer included after the "@" symbol to protect the address from harvesters, while the mailto function does not include the space. The sendemail function should have two parameters. The first is the username portion of the e-mail address and the second is the domain name portion of the e-mail address and is optional. If no domain is supplied, the script will try to use the domain of the current document.

Note in the example above the username ("web-info") is supplied using a string of Unicode characters. While not strictly necessary, using Unicode adds another level of obfuscation to the e-mail address and is recommended. The same applies to the domain, if included.
